# WindOps BC Analyzer

风电运维商业案例智能分析工具 — 基于本地 AI 的成本推断与多方案对比系统。

---

## 版本

| 分支 | 版本 | 说明 |
|------|------|------|
| `main` | **v1.1** | 稳定发布版：规则引擎 + 5方案 + 综合评分 |
| `develop` | v1.1-Dev | 开发版，实验性功能更新 |

---

## 功能简介

将风电运维财务成本记录导入系统后，AI 自动理解 CostGroup 成本分类体系，在工程师创建维修案例时，根据历史数据推断 7 类成本，并生成 **5 种采购方案**（丹麦/中国/本地 × 海运/空运/陆运）的完整成本对比。

### 核心模块

| 模块 | 说明 |
|------|------|
| **AI 配置** | 配置本地 Ollama 地址和模型，一键测试连接 |
| **数据管理** | 上传 CSV/Excel 历史数据，自动映射 CostGroup → 业务成本分类 |
| **案例创建** | 用户输入案例信息，AI 自动填充 7 类成本 |
| **案例审核** | 工程师审查 AI 推断结果，直接修改成本数值 |
| **方案对比** | 5 列并排展示方案，综合评分排名，最优方案高亮推荐 |
| **规则管理** | 46 条预置业务规则，按适用流程自动分流 |

### 技术栈

- **前端**：React 18 + TypeScript + Ant Design 5 + Zustand + Vite
- **后端**：FastAPI + SQLAlchemy 2.0 + SQLite
- **AI**：Ollama 本地部署（推荐 qwen2.5:7b），SSE 流式通信
- **规则引擎**：Python 确定性计算，基于 condition_json 元数据驱动

## 使用流程

1. **AI 配置** — 输入 Ollama 地址，选择模型，测试连接
2. **数据管理** — 上传 Excel/CSV，自动建立 CostGroup 映射
3. **创建案例** — 填写故障信息，点击「创建并 AI 自动填充」
4. **审核成本** — 检查推断结果，可手动修改
5. **生成方案** — 规则引擎秒出 5 方案 + AI 润色，综合评分排名

---

## 规则引擎架构

```
用户审核成本 → 规则引擎(Python) 5方案确定性计算
  → 先保存DB（立即可用）
  → AI 异步润色校验（不阻塞）
  → 前端 5列卡片 + 评分 + 不可行标记
```

**46 条规则**按 condition_json 元数据自动匹配方案参数，用户修改规则后无需改代码。

---

## 常见问题

**Q: 创建案例返回空？** A: 系统内置 fallback 机制，AI 不可用时自动降级为统计计算。

**Q: 模型崩溃？** A: 尝试更小模型如 `qwen2.5:7b` 或 `gemma3:4b`，重启 Ollama。

**Q: 如何添加新采购来源？** A: 在规则管理页添加新规则，填写 source/transport/cost_category 和 rule_value，下次生成方案自动生效。
