# WindOps BC Analyzer

风电运维商业案例智能分析工具 — 基于本地 AI 的成本推断与多方案对比系统。

---

## v1.1-Dev 开发版内容

### 规则引擎（元数据驱动）

基于 46 条预置规则的 Python 确定性计算引擎，按 condition_json 自动匹配方案参数：

```
采购来源(source) × 运输方式(transport) × 目标区域(target_region) × 部件(component)
```

| 维度 | 规则数 | 说明 |
|------|--------|------|
| 零部件成本 | 5 | 丹麦基准×1.0、中国×0.70、发达市场溢价×1.05、发展中折价×0.90、NC远程溢价×1.30 |
| 管理费 | 3 | 丹麦10%、中国3%、本地0% |
| 运输费用 | 13 | 按来源×目的国×运输方式（空运=海运×6.5） |
| 运输天数 | 7 | 基础天数 + 台风季Buffer |
| 罚款权重 | 10 | 11国时效敏感度（0.70~1.30） |
| 历史数据处理 | 6 | 中位数聚合、异常检测、等权处理 |

### 5 方案对比

| 方案 | 来源 | 运输 |
|------|------|------|
| 方案1 | 丹麦总部 | 海运 |
| 方案2 | 丹麦总部 | 空运 |
| 方案3 | 中国子公司 | 海运 |
| 方案4 | 中国子公司 | 空运 |
| 方案5 | 所在国当地 | 陆运 |

Blade + 空运自动标记「不可行」。

### 综合评分

| 因子 | 权重 |
|------|------|
| 总成本（含罚款） | 50% |
| 罚款风险 | 25% |
| 数据置信度 | 15% |
| 方案稳定性 | 10% |

### AI Fallback 机制

AI 不可用时自动降级为统计计算（CostGroup 中位数聚合），确保永不返回空。

### 适用流程规则过滤

| 流程 | 规则数 | 加载方式 |
|------|--------|----------|
| 创建案例 | 6 条（D组） | `applicable_flow = create_case` |
| 生成方案 | 40 条（A/B/C组） | `applicable_flow = generate_plan` |
